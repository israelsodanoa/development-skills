# Evaluation Reference

Definition of done:

- Every acceptance criterion has status and evidence.
- Required commands ran or have explicit waivers.
- Failed checks have attribution, repair, and rerun result.
- Runtime behavior has direct evidence when applicable.
- Approval-required actions are recorded.
- Residual risks are listed.
- Repeated failures or avoidable interventions produce a proposed harness improvement.

Verification report fields:

- task/request ID
- objective
- summary status
- acceptance criteria with status, evidence, and residual risk
- commands
- runtime evidence
- failed checks
- approvals
- residual risks
- harness improvements

Episode traces should summarize actions, commands, context, verification, failures, interventions, permissions, cost signals, outcome, and follow-up controls.
